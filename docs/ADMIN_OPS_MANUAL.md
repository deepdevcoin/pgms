# Admin and Ops Manual

## Owner daily usage

Typical owner use cases:

- review dashboard
- manage PG/property metadata
- inspect layouts and rooms
- assign/deactivate managers
- monitor tenants, complaints, payments, services, notices, and menu

Owner is primarily a supervisory role in the current product direction.

## Manager daily usage

Typical manager use cases:

- onboard tenants
- move tenants between rooms
- archive or deactivate tenant accounts
- maintain room cleaning state
- collect payments and waive fines
- handle complaints
- manage service requests
- manage amenities
- review KYC and request replacements
- approve/reject/manage sublets
- review and process vacate notices
- maintain PG menu

Manager is the operational owner of most day-to-day workflows.

## Tenant daily usage

Typical tenant use cases:

- review dashboard
- pay dues
- raise complaints
- read notices
- request services
- book amenities
- upload KYC
- request sublets
- submit vacate notice
- view menu

## Operational tips

- use manager screens for operational actions, even when owner can still technically inspect related data
- keep KYC and tenant lifecycle in sync
- check room status when completing sublet/vacate actions
- verify notice target audience before publishing

## Common operational mistakes to avoid

- confusing archived tenants with deleted tenants
- treating verified KYC as tenant-editable
- approving sublets before checking date validity
- forgetting that amenities can be maintenance-blocked after bookings exist
